<h3>5.2.4. Searching Systems</h3>

El sistema de búsqueda de MineGuard está diseñado para facilitar el acceso rápido a información crítica dentro de la operación minera. Su propósito es evitar que los usuarios se pierdan entre grandes volúmenes de datos relacionados con vehículos, conductores, alertas, sensores y rutas operativas. Por ello, la búsqueda se adapta a las necesidades de cada plataforma: Web Application, Mobile App y Landing Page.

### Web Application

En la Web Application, el sistema de búsqueda está orientado principalmente al personal del centro de control operativo. Desde módulos como <strong>Gestión de Alertas</strong>, <strong>Flota y Conductores</strong> y <strong>Mapa en Vivo</strong>, los usuarios podrán ubicar información mediante búsquedas y filtros específicos.

Las opciones de búsqueda y filtrado incluirán: tipo de alerta, nivel de criticidad, vehículo involucrado, conductor asignado, ruta operativa, estado del evento, rango horario y estado del vehículo o sensor.

Después de realizar una búsqueda, los resultados se mostrarán mediante <strong>tablas, tarjetas de resumen, indicadores y gráficos</strong>. Por ejemplo, en el registro de alertas en vivo se visualizarán datos como el ID de alerta, nivel, categoría, implicados, ruta, hora y acción disponible. Además, los gráficos del dashboard podrán actualizarse según los filtros seleccionados, permitiendo analizar tendencias de incidentes, eventos de fatiga o conductores en riesgo.

### Mobile App

En la Mobile App, el sistema de búsqueda se enfoca en acciones directas para los conductores u operadores en campo. Según el diseño propuesto, la pantalla de <strong>Selección de Vehículo</strong> incluye un buscador que permite localizar una unidad por nombre o código, como “Camioneta - 04”, “Camioneta - 54” o “Camioneta - 13”.

Los resultados se presentarán mediante <strong>cards</strong>, mostrando información resumida del vehículo, como nombre de la unidad, tipo de vehículo y estado actual. Los estados podrán indicar si el vehículo se encuentra disponible, en uso o en mantenimiento. Esto permite que el usuario identifique rápidamente la unidad que debe utilizar durante su jornada.

### Landing Page</strong>

En la Landing Page no se requiere un sistema de búsqueda avanzado, ya que su objetivo principal es presentar la solución y guiar al visitante por secciones informativas. En este caso, la búsqueda de información se realiza mediante la navegación por etiquetas del menú superior, tales como <strong>Solution</strong>, <strong>How it works?</strong>, <strong>About the team</strong>, <strong>About the product</strong>, <strong>FAQ</strong> y <strong>Sign in</strong>. Estas etiquetas ayudan al usuario a encontrar rápidamente el contenido que necesita sin recorrer toda la página manualmente.

<h3>5.2.5. Navigation Systems</h3>

El sistema de navegación de MineGuard fue diseñado para ofrecer una experiencia clara, rápida y segura, considerando que el producto se utiliza en un contexto operativo donde la información debe ser accesible sin generar confusión. La navegación se adapta según la plataforma utilizada: Landing Page, Web Application y Mobile App. Cada una guía al usuario de acuerdo con sus objetivos específicos dentro del ecosistema digital.

### Landing Page

En la Landing Page, la navegación se realiza mediante una <strong>barra superior horizontal</strong> que contiene el logo de Vertex, el selector de idioma, las secciones principales y el botón de inicio de sesión. Las opciones visibles incluyen <strong>Solution</strong>, <strong>How it works?</strong>, <strong>About the team</strong>, <strong>About the product</strong>, <strong>FAQ</strong> y <strong>Sign in</strong>.

Esta estructura permite que el visitante recorra el contenido de forma ordenada, empezando por la presentación de la solución, continuando con la explicación de su funcionamiento y finalizando con información del equipo, producto o preguntas frecuentes. Además, el botón <strong>Sign in</strong> funciona como una acción directa para acceder al sistema cuando el usuario ya forma parte de la plataforma.

### Web Application

En la Web Application, la navegación se organiza mediante una <strong>barra lateral izquierda</strong>, diseñada para el personal del centro de control operativo. Esta barra agrupa los módulos principales bajo la sección de operaciones, incluyendo <strong>Centro de Control</strong>, <strong>Mapa en Vivo</strong>, <strong>Gestión de Alertas</strong> y <strong>Flota y Conductores</strong>.

La sección activa se resalta visualmente, permitiendo que el usuario identifique en todo momento el módulo en el que se encuentra. Además, en la parte inferior de la barra lateral se muestra información del usuario activo y la opción para cerrar sesión.

Dentro del dashboard web, la navegación interna se apoya en <strong>cards, gráficos, tablas y botones de acción</strong>. Por ejemplo, desde el <strong>Centro de Control</strong> se pueden revisar alertas críticas, eventos de fatiga, vehículos activos y conductores en campo. Asimismo, desde el registro de alertas en vivo, el usuario puede ejecutar acciones como atender una alerta o revisar su estado. Esto permite una navegación contextual, donde el usuario no solo cambia de sección, sino que también interactúa directamente con la información operativa.

### Mobile App

En la Mobile App, la navegación está orientada a conductores y operadores en campo. Se utiliza una <strong>barra de navegación inferior</strong> con accesos principales como <strong>Inicio</strong>, <strong>Desempeño</strong> y <strong>Perfil</strong>. Esta estructura es adecuada para dispositivos móviles, ya que permite acceder rápidamente a las funciones más importantes desde la parte inferior de la pantalla.

En la pantalla de inicio, el usuario puede visualizar su información básica, buscar o seleccionar un vehículo asignado y revisar el estado de las unidades disponibles. La navegación mediante cards permite que el conductor identifique rápidamente cada vehículo y continúe con su flujo operativo sin perder tiempo.

En conjunto, MineGuard utiliza una navegación jerárquica y contextual. La Landing Page guía al visitante en el conocimiento del producto, la Web Application permite al supervisor monitorear y gestionar la operación minera, y la Mobile App facilita que el conductor realice acciones específicas durante su jornada. De esta manera, cada plataforma mantiene una navegación alineada con las metas de sus usuarios y con el contexto de seguridad minera en tiempo real.
